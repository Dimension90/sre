**Deployment:**

replicas: 4: Начальное количество подов, которое справляется с пиковой нагрузкой.
resources: Заданы запросы и лимиты ресурсов для контейнера. Запросы установлены на 128Mi памяти и 100m CPU, лимиты установлены на 256Mi памяти и 500m CPU для обработки первых запросов.
readinessProbe и livenessProbe: Проверки готовности и жизнеспособности подов. initialDelaySeconds: 10 учитывает время инициализации приложения.
startupProbe: Проверка старта приложения с failureThreshold: 30 для учета длительной инициализации.

**HorizontalPodAutoscaler (HPA):**

minReplicas: 2: Минимальное количество подов для ночного времени.
maxReplicas: 10: Максимальное количество подов для пиковой нагрузки.
averageUtilization: 50: Масштабирование на основе использования CPU.

**PodDisruptionBudget (PDB):**

minAvailable: 2: Обеспечение минимального количества доступных подов во время обновлений или удалений.
